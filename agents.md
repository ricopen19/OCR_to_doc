# Agents / Roles

## 1. 関係者
- **開発者（本人）**: プロジェクト全体の設計・実装・運用。個人開発。
- **Codex（AI アシスタント）**: コード編集補助、ドキュメント整備、設計ディスカッションの相棒。並列開発は将来的に検討。

## 2. コラボレーション方針
- 作業開始前に `docs/context_engineering.md` を参照して目的・制約を共有。
- README や `docs/Tasks.md` に沿ってタスクを選定し、変更内容・対象ファイルを明記する。
- 作業の区切りごとに各ドキュメントの内容を最新化する。

## 3. 注意事項 / Open Questions
### 機能実装の方針
- ある機能を実装する前に、どのような方針で具体的に何を実装するか分かりやすく説明する。
- 実装する際の考慮事項や懸念事項を確認・共有した上で、最終的に実装するかどうか判断する。
- 無闇に機能を増やさず、よりシンプルな解決方法を優先・提案する
- 「実装して」と言うまでは勝手に実装しない
- 機能の改善の見通しが立たない場合やリスクが伴う場合は、別の角度からアプローチを試みる。

### コマンドラインで実行するコードの出力形式
- 実行するコードは一度に実行できるように改行してコマンドブロックで出力する。
例:
コマンド1
```bash
poetry run python dispatcher.py sample.heic \
  --mode full \
  --figure \
  --fallback-tesseract
```
